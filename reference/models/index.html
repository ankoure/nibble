    Models¶

    Core immutable data structures that flow through the nibble pipeline.

    nibble.models ¶

    Core immutable data structures flowing through the nibble pipeline.

    Position dataclass ¶

    GPS position reported by a vehicle.

    latitude/longitude are WGS84 decimal degrees. bearing is degrees clockwise from north (0-359). speed is meters per second. Both bearing and speed may be absent from the feed.

    Attributes:

    Name Type Description
    latitude float

    WGS84 decimal degrees.
    longitude float

    WGS84 decimal degrees.
    bearing float | None

    Heading in degrees clockwise from north (0-359), or None if the feed does not report it.
    speed float | None

    Speed in meters per second, or None if not reported.

    VehicleEvent dataclass ¶

    Immutable snapshot of a vehicle's state at a point in time.

    Every event carries two quality tags:

    provenance - how the event was produced:

    • "observed" - directly reported by the GTFS-RT feed
    • "inferred" - position observed, but trip info carried forward from a prior poll
    • "interpolated" - synthetic event generated to fill a stop gap between polls
    • "manual" - trip assignment issued by an operator via the corrections API

    confidence - certainty about the vehicle's trip assignment:

    • "confirmed" - vehicle reported a trip_id in this poll
    • "inferred" - trip_id carried forward; vehicle may have changed trips
    • "stale" - vehicle exceeded the stale threshold; a remove event follows

    Attributes:

    Name Type Description
    vehicle_id str

    Stable identifier for the vehicle (from the GTFS-RT feed).
    position Position

    Current GPS position.
    timestamp datetime

    UTC timestamp of this observation.
    provenance Literal['observed', 'inferred', 'interpolated', 'manual']

    How this event was produced ("observed", "inferred", "interpolated", or "manual").
    confidence Literal['confirmed', 'inferred', 'stale']

    Certainty about the trip assignment ("confirmed", "inferred", or "stale").
    trip_id str | None

    GTFS trip identifier, or None if unknown.
    route_id str | None

    GTFS route identifier, or None if unknown.
    stop_id str | None

    Current stop identifier, or None if unknown.
    current_stop_sequence int | None

    Stop sequence number within the current trip, or None if unknown.
    current_status Literal['INCOMING_AT', 'STOPPED_AT', 'IN_TRANSIT_TO']

    Vehicle status relative to the current stop - "INCOMING_AT", "STOPPED_AT", or "IN_TRANSIT_TO".
    direction_id int | None

    GTFS direction (0 or 1), or None if unknown.
    label str | None

    Human-readable vehicle label (e.g. bus number), or None.

    SSEEvent dataclass ¶

    An SSE envelope ready for broadcast to clients.

    Attributes:

    Name Type Description
    event_type Literal['reset', 'add', 'update', 'remove']

    The SSE event name - "reset", "add", "update", or "remove".

    • "reset" - full snapshot of all known vehicles; sent once to new subscribers; data is a list of serialized vehicle dicts (JSON:API format).
    • "add" - a vehicle has appeared in the feed for the first time since the last reset; data is a single serialized vehicle dict (JSON:API format).
    • "update" - a vehicle already in the feed changed state; data is a single serialized vehicle dict (JSON:API format).
    • "remove" - a single vehicle has left the feed; data is {"id": ...}.
    data list[dict[str, Any]] | dict[str, Any]

    A single serialized vehicle dict for "add"/"update"/"remove" events, or a list of vehicle dicts for "reset" events.

    Trip dataclass ¶

    Static GTFS trip metadata, loaded from trips.txt.

    Attributes:

    Name Type Description
    trip_id str

    Unique GTFS trip identifier.
    route_id str

    Route this trip belongs to.
    direction_id int | None

    Travel direction (0 or 1), or None if absent.
    shape_id str | None

    Associated shape identifier, or None if absent.

    StopTime dataclass ¶

    Scheduled stop entry from stop_times.txt.

    arrival_time and departure_time are GTFS HH:MM:SS strings. Hours may exceed 23 for service running past midnight (e.g. "25:30:00").

    Attributes:

    Name Type Description
    trip_id str

    Trip this stop time belongs to.
    stop_id str

    Stop being served.
    stop_sequence int

    Ordering of the stop within the trip.
    arrival_time str | None

    Scheduled arrival as a GTFS HH:MM:SS string, or None.
    departure_time str | None

    Scheduled departure as a GTFS HH:MM:SS string, or None.
    shape_dist_traveled float | None

    Distance along the shape from the first stop to this stop, in the units used by the feed's shapes.txt, or None if not provided.
